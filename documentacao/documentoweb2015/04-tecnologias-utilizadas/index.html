  • Angular 2


    Angular 2 é um framework mantido pelo Google nos auxilia a abstrair a manipulação do DOM da lógica do aplicativo, abstrair o acoplamento entre o lado cliente e o lado servidor da aplicação e a melhorar a qualidade dos testes.
    Angular é basicamente um conjunto de bibliotecas (algumas core, outras não) que trabalham em cooperação.
    O framework segue o padrão MVC (Model View Controller) e presa pelo baixo acoplamento entre as camadas de apresentação, dados e componentes lógicos. Utilizando de injeção de dependência ele traz alguns componentes que normalmento são utilizados no lador servidor para o lado cliente deixando, consequentemente, o backend mais leve.
    Abaixo segue a arquitetura básica do Angular 2:

    Aplicações Angular são desenvolvidas pela composição de templates HTML enriquecido com anotações e marcações angular. Para manipular os templates escrevemos componentes e criamos serviços para executar a lógica da aplicação. Componentes e serviços são emapcotados em módulos.
    Componentes nada mais são que uma classe com a anotação @Componente, exemplo de componente: 
    @Component({
						selector:'hero-list',
						templateUrl:'app/hero-list.component.html',
						providers:   [ HeroService ]
					})
					export class HeroListComponent implements OnInit {
					/* . . . */
					}

  • Ionic 2


    Ionic framework é uma SDK (Software Development Kit) de código aberto que possibilita ao desenvolvedor de aplicações web que já possui conhecimento em HTML5, CSS e JavaScript a criar aplicações mobile.
    Ionic foca principalmente na interação com a interface do aplicativo, ele visa simplificar o desenvolvimento do front-end. Basicamente ele preenche a lacuna do HTML5 para o desenvolvimento nativo do aplicativo.
    Além do framework permitir que aplicações mobile sejam criadas praticamente do mesmo modo de uma aplicação web comum ainda permite que os testes sejam realizados diretamente pelo browser.

    Para saber mais acesse: Ionic 2.
